# 三湘问道 v2.0 → v3.0 升级建议

**创建日期**: 2026-01-19
**目的**: 基于knowledge base分析,补充v2.0缺失的核心能力

---

## 🎯 核心问题诊断

### v2.0的3大断层

#### 断层1: 缺少多维对标引擎 ❌

**现状**:
- Step 3只是简单列出对标对象
- Step 4只是搜索理论文档
- 没有CEO思维模式对标
- 没有主动推理分析
- 没有发散思维触发器

**影响**:
- 对标分析流于表面
- 缺少深度洞察
- 无法生成"CEO会怎么做"的建议

#### 断层2: Evidence-Based系统不完整 ⚠️

**现状**:
- 有"搜索知识库"要求
- 但没有证据等级系统(A/B/C)
- 没有失败模式(FM)分类
- 没有可证伪点强制要求

**影响**:
- 评估结论可信度不明
- 缺少系统化风险预警
- 无法强制执行evidence-based

#### 断层3: 失败案例系统缺失 ❌

**现状**:
- 只提到"反面案例警示"
- 但没有失败案例库
- 没有FM标签系统
- 没有"同模式失败者"对比

**影响**:
- 只看成功,容易盲目乐观
- 风险识别不够深入
- 缺少"为什么别人失败了"的警示

### 升级模块2: Evidence-Based系统强化

**新增: 证据等级系统**

每条关键断言必须标注证据等级:

```yaml
证据等级系统:
  A级_直接数据:
    - 官方财报
    - 第三方审计报告
    - 政府公开数据
    - 用户公开评论(大样本)
    可信度: 90%+
    
  B级_间接推断:
    - 行业报告
    - 媒体报道(权威媒体)
    - 专家访谈
    - 竞品对比
    可信度: 70-90%
    
  C级_合理推测:
    - 逻辑推理
    - 类比其他案例
    - 模型估算
    可信度: 50-70%
    
  X级_无法验证:
    - 创始人宣称(无证据)
    - 小道消息
    - 纯主观判断
    可信度: <50% (标红预警)
```

**使用方式**:

```markdown
## 示例: 用户需求分析

**断言1**: "目标用户有强烈痛点" [证据等级: B]
- 证据: 行业报告显示60%用户有此需求
- 来源: [报告名称]
- 可信度: 75%

**断言2**: "用户愿意付费" [证据等级: X]
- 证据: 创始人说"用户很满意"
- ⚠️ 警告: 缺少付费转化数据,可信度<50%
- 建议: D30验证付费意愿
```

## 🎯 总结

### v2.0的3大断层

1. **缺少多维对标引擎** → CEO模式对标、主动推理、发散思维
2. **Evidence-Based不完整** → 证据等级、FM标签、可证伪点
3. **失败案例系统缺失** → 失败案例库、FM预警、反面对比

### v3.0的3大核心升级

1. **主动推理引擎** (5层推理,20分钟)
2. **证据等级系统** (A/B/C/X,强制标注)
3. **失败案例对比** (至少1个,FM标签)

### 实施路径

- **Phase 1** (本周): 最小化升级,+20分钟,质量提升50%
- **Phase 2** (下周): 完整v3.0,+45分钟,质量接近专业VC
- **Phase 3** (4周): 案例库冷启动,30-50个案例

### 最重要的一句话

**v2.0是"评估框架"**
**v3.0是"评估框架 + 主动推理引擎 + 失败模式预警"**

→ 从"打分工具" → 升级为 → "投资级分析系统"
