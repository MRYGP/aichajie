# 可复用模式库

## 简介

本库收录了从成功案例中提炼的可复用模式，用于AI产品分析时的模式识别和迁移。这些模式是：

- 跨案例的共性规律
- 可复用的成功公式
- 经过验证的打法
- 可迁移到其他场景

## 模式分类

### 增长模式

| 模式 | 核心机制 | 适用场景 | 案例 |
|------|---------|---------|------|
| PLG（产品驱动增长） | 产品本身=营销渠道 | toB SaaS | Notion、Slack、Figma |
| 病毒传播 | 用户自发邀请 | 社交/协作产品 | Dropbox、Loom |
| 内容营销 | 免费内容教育用户 | 工具/平台 | HubSpot、Buffer |
| 社区驱动 | 社区>产品 | 开发者/设计师 | GitHub、Figma |

### 商业模式模式

| 模式 | 核心机制 | 适用场景 | 案例 |
|------|---------|---------|------|
| Freemium | 免费+增值服务 | 工具/平台 | Notion、Slack、Canva |
| 订阅制 | 持续付费 | 内容/服务 | Netflix、Spotify |
| 平台抽佣 | 双边市场+抽佣 | 平台型产品 | Airbnb、Shopify |
| API优先 | 开发者生态 | 基础设施 | Stripe、Twilio |

### 产品策略模式

| 模式 | 核心机制 | 适用场景 | 案例 |
|------|---------|---------|------|
| 边缘用户突围 | 从被忽视的用户切入 | 新市场 | Perplexity、拼多多 |
| 设计民主化 | 降低使用门槛 | 专业工具 | Canva、Figma |
| 极简主义 | 只做核心功能 | 工具类产品 | Linear、Arc |
| 协作优先 | 多人协作>单人使用 | 生产力工具 | Notion、Figma |

## 使用方法

### 步骤1：识别产品模式

分析产品使用了哪些模式：
- 增长模式：如何获客？
- 商业模式：如何赚钱？
- 产品策略：如何差异化？

### 步骤2：提炼模式

1. 打开本README，浏览模式列表
2. 找到产品使用的模式（可能多个）
3. 阅读模式的核心机制
4. 对比产品是否符合

### 步骤3：迁移应用

1. 理解模式的适用条件
2. 判断你的场景是否符合
3. 设计迁移方案
4. 验证可行性

### 步骤4：组合创新

多个模式可以组合，形成新的打法：

**示例**：
- PLG + Freemium = Notion
- 边缘用户突围 + 设计民主化 = Canva
- 协作优先 + API优先 = Figma

## 模式提炼方法

### 方法1：跨案例对比

1. 选择3-5个成功案例
2. 找出共同特征
3. 提炼为模式
4. 验证边界条件

### 方法2：失败案例反推

1. 选择失败案例
2. 分析失败原因
3. 提炼"避免模式"
4. 设计"成功模式"

### 方法3：理论映射

1. 从商业理论中提炼模式
2. 用案例验证
3. 总结适用条件

## 示例

**分析一个AI工具产品**：

1. **识别模式**：
   - 增长：PLG（用户用了就推荐）
   - 商业模式：Freemium（个人免费+团队付费）
   - 产品策略：极简主义（只做核心功能）

2. **提炼模式**：
   - 主要模式：PLG + Freemium + 极简主义
   - 对标案例：Notion、Linear

3. **迁移应用**：
   - 如果要做类似产品：
     - Month 1-3：MVP上线，个人免费
     - Month 4-6：优化分享机制（PLG）
     - Month 7-12：添加团队功能（Freemium转化）

4. **组合创新**：
   - 可以加入AI功能（Notion没有的）
   - 形成：PLG + Freemium + 极简主义 + AI增强
