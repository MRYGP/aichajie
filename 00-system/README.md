## 🚀 版本历史

### v3.1 (2026-02-10) 🔄 业务闭环升级（测试中）

**核心升级**:
- ✅ 新增 Pre-Step 产品收集引导（10 维度产品池）
- ✅ 新增 Step 8 商业化决策（复制 / 工具 / 代理 / 观察 四路决策树）
- ✅ 新增周度节奏引导（周一收集→周五决策→月末评审）
- ✅ 打通「收集→拆解→决策→执行」完整业务闭环

**当前状态**:
- 作为 **最新版（测试中）** 提供，建议小范围试用 1-2 周
- 确认稳定后，可逐步替换 v3.0 成为默认稳定版

---

### v3.0 (2026-01-19) ⭐ 重大升级

**核心升级**:
- ✅ **多维对标引擎**: CEO思维模式对标 + 主动推理 + 发散思维
- ✅ **Evidence-Based系统**: 证据等级(A/B/C/X) + FM标签
- ✅ **失败案例系统**: 失败模式预警 + 反面对比

**新增功能**:
- Step 3.5: CEO思维模式对标 (15分钟)
- Step 3.6: 主动推理分析 (20分钟，5层推理)
- Step 3.7: 发散思维检查 (10分钟)
- 证据等级强制标注
- 失败模式(FM)预警系统
- 90天可证伪验证计划

**时长变化**:
- 快速评估: 30分钟 (原7步)
- 标准评估: 90分钟 (7步 + 前3层推理) ⭐推荐
- 深度评估: 115分钟 (完整7+3步)

**质量提升**: 预计50-100%

## 📊 版本选择指南

### 使用 v3.0（推荐｜当前稳定版）

✅ **适合场景**:
- 需要深度分析的重点项目
- 需要投资级分析报告
- 需要系统化风险识别
- 需要"CEO会怎么做"的建议

✅ **核心优势**:
- 主动推理引擎（5层推理）
- CEO思维模式对标
- 失败模式预警
- 证据等级系统

⚠️ **时长要求**: 90-115分钟

---

> 说明：v3.1 为在 v3.0 之上叠加「产品收集 + 商业化决策 + 周度节奏」的增强版，建议先在少量项目中测试 1-2 周，验证体验和稳定性后，再考虑将 v3.1 作为默认稳定版，并视情况是否保留或归档 v3.0。

### 使用 v3.1（体验最新版｜测试中）

✅ **适合场景**:
- 你已经熟悉 v3.0，希望连同「产品收集 → 评估 → 商业化决策 → 执行」一整条链路一起跑
- 需要在一个系统指令里同时解决「去哪找项目」「评估完怎么做」「每周怎么安排」

✅ **核心优势**:
- 在 v3.0 全能力基础上，增加：
  - Pre-Step 产品收集模块（10 维度）
  - Step 8 商业化决策（复制 / 工具 / 代理 / 观察）
  - 顶部周度节奏与月度 KPI 引导

⚠️ **当前状态**:
- 标记为「最新版（测试中）」：建议用于试点项目和内部自用
- 如果在 1-2 周测试期内未发现明显问题，可在后续：
  - 将 v3.1 提升为「推荐｜稳定版」
  - 再评估是否仅保留 v3.1，或以「历史版本」形式保留 v3.0

## 🎯 快速开始

### 使用 v3.0（推荐流程｜稳定版）

1. **创建Claude Project**
   - 打开 [Claude.ai](https://claude.ai)
   - 创建新Project："三湘问道-AI项目评估"

2. **导入系统指令**
- 复制 `claude-project-instructions-v3.0.md` 到 Custom Instructions（稳定版推荐）
   - 等待加载完成

3. **上传知识库**
   - 上传 `01-knowledge-base/` 下的所有文档
   - 上传 `taxonomy/` 下的YAML文件
   - 等待处理完成

4. **开始评估**
   ```
   帮我评估一个AI项目：
   
   项目名称：[项目名]
   定位：[一句话说明]
   阶段：[idea/MVP/...]
   团队：[X人，背景：...]
   商业模式：[如何赚钱]
   
   请用标准评估模式（推荐）
   ```

5. **查看报告**
   - 报告自动保存到 `02-evaluation-reports/`
   - 使用模板：`templates/deep-evaluation-template.md`

---

### 使用 v3.1（测试流程｜最新版）

1. **创建 Claude Project**
   - 建议单独新建一个 Project，用于 v3.1 试点

2. **导入系统指令**
   - 复制 `claude-project-instructions-v3.1.md` 到 Custom Instructions
   - 注意说明：「这是在 v3.0 上增加收集与决策模块的测试版」

3. **选取少量重点项目试跑**
   - 从产品清单中选 3-5 个项目
   - 完整跑一遍：Pre-Step → 7+3 步 → Step 8

4. **记录问题与改进点**
   - 使用 `00-system/质量检查指南.md` 作为检查表
   - 特别关注：触发条件是否清晰、节奏引导是否好用、四路决策是否够直观

> 建议：v3.1 测试期为 1-2 周，测试通过后再更新本 README 中的推荐版本说明，并评估是否需要长期保留 v3.0 作为历史稳定版。

## ⚠️ 重要提示

### v3.0使用注意事项

1. **证据等级标注是强制要求**
   - 所有关键断言必须标注[A/B/C/X]
   - X级证据会自动标红警告

2. **主动推理不可跳过**
   - 至少完成5 Whys + 反事实推理
   - Level 5可证伪设计是强制要求

3. **失败案例必须引用**
   - 每个评估至少1个失败案例
   - 必须标注FM标签
   - 必须对比分析

4. **知识库必须完整**
   - 缺少文档会影响分析质量
   - 定期更新案例库
