## 📊 证据等级系统（强制要求）

### 证据分级标准

每个关键断言必须标注证据等级：

**[A级 - 直接数据]** 可信度90%+
- 官方财报、第三方审计、政府数据、用户公开评论(大样本)

**[B级 - 间接推断]** 可信度70-90%
- 行业报告、权威媒体、专家访谈、竞品对比

**[C级 - 合理推测]** 可信度50-70%
- 逻辑推理、类比案例、模型估算

**[X级 - 无法验证]** 可信度<50%
- 创始人宣称(无证据)、小道消息、纯主观判断
- ⚠️ 自动标红预警

### 使用示例

```markdown
**断言1**: "目标用户有强烈痛点" [证据等级: B]
- 证据: 行业报告显示60%用户有此需求
- 来源: [报告名称]
- 可信度: 75%

**断言2**: "用户愿意付费" [证据等级: X] ⚠️
- 证据: 创始人说"用户很满意"
- 警告: 缺少付费转化数据，可信度<50%
- 建议: D30验证付费意愿
```

### 1. 主动推理引擎 (⭐⭐最重要)

**以前**: 只告诉你"这个项目可能有风险"

**现在**: 告诉你:
- 为什么会有这个风险？(5 Whys追根溯源)
- 如果没有某个因素会怎样？(反事实推演)
- 这个模式在哪些场景下失效？(模式识别+边界)
- 90天内如何验证？(可证伪设计)

**举例**:
```
以前: "这个项目依赖流量红利，有风险"

现在: 
Why 1: 用户从哪来？→ 90%来自抖音
Why 2: 为什么选抖音？→ 2024年抖音算法红利期
Why 3: 红利能持续吗？→ 历史上红利期6-18个月
Why 4: 如果红利结束？→ CAC从$5暴涨到$50
Why 5: 有备选方案吗？→ 没有测试其他渠道

反事实: 如果抖音红利结束，CAC>$50，项目崩溃

可证伪: D60检查CAC，如果>$20立即Kill
```

### 2. 证据等级系统

**以前**: "创始人说用户很满意"

**现在**: 
```
"用户很满意" [X级] ⚠️ 
- 证据: 创始人宣称
- 可信度: <50%
- 建议: D30验证付费意愿和续费率
- 警告: 不可作为决策依据
```

**分级标准**:
- **[A级]**: 官方数据，可信度90%+ (财报、审计)
- **[B级]**: 行业报告，可信度70-90% (IDC、Gartner)
- **[C级]**: 合理推测，可信度50-70% (需要验证)
- **[X级]**: 无法验证，可信度<50% (自动标红)

## ⚠️ 常见问题

### Q1: v3.0需要更多时间，值得吗？

**A**: 值得。质量提升50-100%，时间只增加30-40分钟。

**举例**:
- v2.0: "这个项目可能有风险" (无证据)
- v3.0: "这个项目有FM001风险，类似XX项目在6个月后因CAC暴涨倒闭。建议D60检查CAC，如果>$20立即Kill" (有证据+有对策)

---

### Q2: 证据等级标注很麻烦吗？

**A**: 不麻烦。系统会自动提示你标注，只需要标注关键断言。

**举例**:
```
用户增长快 → [需要证据]
用户增长率120% (来源: 官方数据) [A级] → [合格]
```

---

### Q3: 如果没有失败案例怎么办？

**A**: 系统会从knowledge base搜索相似案例。如果确实没有，标注"缺少失败案例参考"。

---

### Q4: CEO对标是什么意思？

**A**: 分析"如果Musk/Jobs/Bezos做这个项目，会怎么做？"

**举例**:
```
项目: AI代码助手

CEO对标: Elon Musk
- 匹配点: 技术驱动 ✓
- 不匹配点: 没有疯狂目标
- Musk会做: 目标不是"帮助写代码"，而是"消灭手写代码"
- 建议: 提升目标，从"辅助"到"替代"
```
