# 文档生成规范

> **用途**：把 Claude 的结构化要点生成标准化 Markdown 文档并落盘，同时同步更新索引  
> **原则**：Cursor 负责"工程化落盘"，不重新发明分析结论

---

## 生成规则（强制）

### 1) 文件命名

- **案例文档**：`<企业名>-<案例关键词>.md`
  - 示例：`Tesla-创办电动汽车公司.md`
  - 示例：`Amazon-三大支柱业务与飞轮效应.md`
  
- **模型文档**：`<模型名称>.md`
  - 示例：`飞轮效应.md`
  - 示例：`网络效应.md`
  
- **洞察文档**：`洞察XX-<标题>.md` 或 `趋势XX-<标题>.md`
  - 示例：`洞察01-订阅模式的三个关键变量.md`
  - 示例：`趋势01-AI搜索的演进路径.md`
  
- **评估报告**：`<项目名>-评估报告-YYYYMMDD.md`
  - 示例：`Perplexity-评估报告-20260117.md`

### 2) 元信息必填

每个文档顶部必须包含：
- 标题（H1）
- 日期（拆解/创建/提出）
- 来源（可追溯）
- 标签（如适用）

### 3) 口径统一

- 所有"应用/启发"统一写作：**通用启发 / 可迁移原则**（带边界与最小验证动作）

## 同步更新清单（每次生成后必须执行）

### 新增/更新案例

- [ ] 保存到 `01-knowledge-base/cases/` 或 `04-case-library/`
- [ ] 更新 `04-case-library/case-index.md`（如果存在）
- [ ] 如引用新模型：创建模型文件并更新相关索引
- [ ] 如产生洞察：创建洞察文件并更新相关索引

### 新增/更新模型

- [ ] 保存到 `01-knowledge-base/methods/`
- [ ] 在相关案例中补充"模型标签/相关链接"
- [ ] 更新 `01-knowledge-base/README.md`（如果维护了模型列表）

### 新增/更新洞察

- [ ] 保存到 `01-knowledge-base/methods/`
- [ ] 在支撑案例中补充"洞察引用/相关链接"
- [ ] 更新 `01-knowledge-base/README.md`（如果维护了洞察列表）

### 新增/更新评估报告

- [ ] 保存到 `02-evaluation-reports/YYYY-QX/`
- [ ] 更新 `04-case-library/case-index.md`（如果适用）
