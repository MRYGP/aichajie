# 思维模型库

## 简介

本库收录了10个核心思维模型，用于AI产品分析时的主动推理。这些模型来自不同领域（工程、心理学、经济学等），帮助：

- 深度归因分析（找到根本原因）
- 反事实推理（验证假设）
- 模式识别（提炼共性）
- 机制设计（设计飞轮）
- 可证伪验证（设计实验）

## 使用方法

### 步骤1：选择合适模型

根据分析目标选择模型：

**归因分析** → 5 Whys
**验证假设** → 反事实推理
**本质思考** → 第一性原理
**设计机制** → 机制设计
**验证实验** → 可证伪设计
**避免失败** → 逆向思维
**多元分析** → 思维模型清单
**需求分析** → Jobs to be Done
**商业模式** → 单位经济模型
**护城河分析** → 网络效应分析

### 步骤2：阅读模型文档

1. 打开对应的模型文档
2. 阅读"模型简介"和"使用场景"
3. 理解"操作步骤"
4. 查看"实战案例"

### 步骤3：应用模型分析

1. 按照"操作步骤"执行
2. 记录每个步骤的输出
3. 参考"常见错误"，避免踩坑
4. 使用"检查清单"自检

### 步骤4：组合使用

多个模型可以组合使用，形成更强的分析能力：

**示例组合**：
- 5 Whys（找根因） + 反事实推理（验证）
- 第一性原理（本质思考） + 机制设计（设计方案）
- 逆向思维（避免失败） + 可证伪设计（验证方案）

## 示例

**分析Perplexity成功原因**：

1. **选择模型**：5 Whys（归因分析）

2. **打开文档**：阅读`five-whys.md`

3. **执行分析**：
   ```
   现象：Perplexity获得100万用户
   
   Why 1：为什么用户选择Perplexity？
   → 因为它直接给答案，不用点10个链接
   
   Why 2：为什么用户不想点10个链接？
   → 因为太浪费时间，90%的链接都没用
   
   Why 3：为什么Google不直接给答案？
   → 因为Google的商业模式是广告，需要用户点链接
   
   Why 4：为什么AI可以直接给答案？
   → 因为LLM可以理解语义，生成答案
   
   Why 5：为什么现在才有这个产品？
   → 因为LLM（GPT-3.5）到2022年才成熟
   
   根因：时间窗口（LLM成熟）+ 用户需求（直接答案）+ 商业模式创新（订阅制）
   ```

4. **组合使用**：用反事实推理验证
   - 如果LLM没成熟，Perplexity会成功吗？→ 不会
   - 如果Google也做AI搜索，Perplexity还有机会吗？→ 有，因为商业模式不同

## 模型难度说明

- ⭐⭐：简单，15-30分钟掌握
- ⭐⭐⭐：中等，需要1-2小时练习
- ⭐⭐⭐⭐：较难，需要深度理解
- ⭐⭐⭐⭐⭐：很难，需要大量实践
