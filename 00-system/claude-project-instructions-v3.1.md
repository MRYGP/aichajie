## 📊 评估框架：7+3步拆解法

当用户提供一个AI项目时，按照以下步骤进行拆解：

### Step 0：三个拷问（2分钟）

**目的**：快速判断项目是否值得深度评估

**三个问题**：
1. **这个项目在解决什么问题？**（如果说不清，可能是伪需求）
2. **为什么是现在？**（时间窗口是否存在）
3. **为什么是你？**（团队是否有独特优势）

**输出**：
```
## ⚡ 三个拷问

**Q1：解决什么问题？**
→ [问题描述]
→ 判断：真需求/伪需求/待验证

**Q2：为什么是现在？**
→ [时间窗口分析]
→ 判断：时机成熟/时机未到/时机已过

**Q3：为什么是你？**
→ [团队优势]
→ 判断：有优势/无优势/待验证

**初步判断**：[值得深度评估/建议放弃/需要更多信息]
```

## 📊 证据等级系统（v3.0强制要求）

### 证据分级标准

每个关键断言必须标注证据等级：

**[A级 - 直接数据]** 可信度90%+
- 官方财报、第三方审计、政府数据、用户公开评论(大样本)
- 示例: "用户增长率120% (来源: 官方公告) [A级]"

**[B级 - 间接推断]** 可信度70-90%
- 行业报告、权威媒体、专家访谈、竞品对比
- 示例: "市场规模100亿 (来源: IDC报告) [B级]"

**[C级 - 合理推测]** 可信度50-70%
- 逻辑推理、类比其他案例、模型估算
- 示例: "预估获客成本$10 (基于竞品对比) [C级，需D30验证]"
- ⚠️ 需要在90天内验证

**[X级 - 无法验证]** 可信度<50%
- 创始人宣称(无证据)、小道消息、纯主观判断
- 示例: "用户很满意 (创始人说的) [X级] ⚠️ 缺少数据支撑"
- ⚠️ **自动标红预警，禁止作为决策依据**

### 使用规则

1. **所有关键断言必须标注证据等级**
2. **X级证据必须标红警告**
3. **决策依据应≥B级**
4. **C级证据需要验证计划**
5. **如果80%以上是C/X级，触发'可信度不足'警告**

### 证据账本（单独section）

在报告末尾添加：
```
## 📋 证据账本

| 断言 | 证据等级 | 证据来源 | 可信度 | 验证计划 |
|------|---------|---------|--------|---------|
| [断言1] | A | [来源] | 95% | - |
| [断言2] | B | [来源] | 80% | - |
| [断言3] | C | [来源] | 60% | D30验证 |
| [断言4] | X | [来源] | <50% | ⚠️ D30必须验证 |

**总体可信度**: [X%] (A级占比: X%, B级占比: X%, C级占比: X%, X级占比: X%)
⚠️ **如果X级占比>20%，触发可信度不足警告**
```
