# 商业理论库

## 简介

本库收录了10个经典商业理论，用于AI产品分析时的理论对标。通过将产品特征与经典理论匹配，可以：

- 理解产品的底层逻辑
- 预测产品的发展路径
- 识别理论边界和适用条件
- 提炼可应用的原则

## 使用方法

### 步骤1：识别产品特征

分析产品的以下维度：
- 目标用户（主流 vs 边缘）
- 产品定位（功能多 vs 功能少）
- 定价策略（高价 vs 低价/免费）
- 增长方式（烧钱 vs 自然增长）
- 商业模式（订阅 vs 广告 vs 平台）

### 步骤2：匹配相关理论

1. 打开本README，浏览10个理论的核心概念
2. 根据产品特征，初步判断可能匹配的理论（2-3个）
3. 打开对应的理论文档，阅读"匹配方法"部分
4. 使用"匹配信号"检查产品是否符合

### 步骤3：验证符合度

1. 使用"验证问题"部分，对产品进行深度分析
2. 记录符合的部分和不符合的部分
3. 分析不符合的原因：是理论不适用，还是产品策略有问题？
4. 推荐阅读：基于产品短板，推荐读什么书？

### 步骤4：提炼可应用原则

1. 阅读"可应用原则"部分
2. 结合产品实际情况，设计具体行动方案
3. 识别边界条件：这个理论在什么情况下失效？

## 示例

**分析Notion时**：

1. **初步判断**：
   - 从个人用户切入（边缘市场）→ 破坏式创新
   - Freemium模式 → 精益创业
   - 社区驱动增长 → 增长黑客

2. **打开文档**：阅读`disruptive-innovation.md`的"匹配方法"

3. **验证检查**：
   - ✅ 信号1：目标用户是边缘（个人用户，vs Confluence的企业用户）
   - ✅ 信号2：功能更灵活（vs Confluence的固定模板）
   - ✅ 信号3：价格更低（个人免费，vs Confluence的$$$）

4. **深度分析**：使用"验证问题"，发现符合度85%

5. **输出结论**：
   - 主要理论：破坏式创新
   - 可应用原则：先站稳低端（个人用户），再向上渗透（企业用户）
   - 推荐阅读：《创新者的窘境》第3章

## 理论组合分析

一个产品可能同时符合多个理论，形成"理论组合"：

**示例：Perplexity**
- 破坏式创新（从边缘用户切入）
- 精益创业（快速迭代）
- 蓝海战略（重新定义搜索）

**组合效应**：1+1+1>3，形成更强的竞争优势
