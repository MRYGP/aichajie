# 三湘问道 v3.2 升级包 - 执行总结

## 🚨 先看这个：防误用警告

### 警告1：本框架不是"成功预测器"

❌ **错误理解**："反共识强度高 → 项目成功率高"
✅ **正确理解**："反共识是赔率信号（high payoff），不是概率信号（high probability）"

**真相**：
- 大多数反共识项目会失败（因为大多数反共识是错的）
- 少数成功的反共识项目会有巨大回报（right tail）
- **必须同时检查**：可证伪 + 楔子清晰

### 警告2：历史不会完全重复

❌ **错误理解**："对标Google，我们也能成"
✅ **正确理解**："学习Google的可迁移规律，避免生搬硬套"

**必须考虑时代差异**：
- 2010年的技术成本 vs 2025年
- 2010年的监管环境 vs 2025年
- 2010年的竞争格局 vs 2025年

### 警告3：分数只能用于排序，不能单独决策

❌ **错误理解**："8.5分，立项！"
✅ **正确理解**："8.5分，核心假设是X，D30验证Y指标"

**本框架输出的是**：
- 关键假设清单
- 证据账本
- 验证计划

**不输出**：
- 成功概率
- 投/不投的二元决策

## 📋 D30验收标准（可证伪）

如果你开始执行v3.2升级，**D30必须达到以下标准**：

**测试方法**：

- 检查是否生成了D30预测+Kill阈值

### P0-2：案例库质量标准

**验收标准**：
- [ ] 每个成功案例必须有失败对照
- [ ] 每个失败案例必须有D30预警信号
- [ ] 否则案例不计入库

**测试方法**：
- 随机抽3个案例检查
- 是否符合模板要求

### P0-3：盲测校准（预警命中率≥50%）

**验收方法**：
- 找10个你不知道结局的早期项目（或假装不知道）
- 用框架输出"关键假设+FM+验证计划"
- 对照真实结果

**验收标准**：
- [ ] 是否提前指出了最终失败的那条关键假设？
- [ ] 命中率≥50%算及格（不是预测成功，是预警失败）

**为什么是50%？**
- 不追求100%命中（那是过拟合）
- 50%意味着"比随机猜测好一倍"
- 重点是"识别失败假设"，不是"预测成功"

## 🎯 正确vs错误的使用姿势

### ❌ 错误姿势（会害人）

1. **只看反共识，不看可证伪**
   - "这个idea太反共识了，肯定能成！"
   - 忽略：能在D30验证吗？

2. **生搬硬套历史案例**
   - "Google当年也是2个人，我们也能成！"
   - 忽略：时代差异、资源差异

3. **把分数当决策依据**
   - "8.5分，立项！不用验证了！"
   - 忽略：假设验证、证据等级

### ✅ 正确姿势（安全有效）

1. **反共识+可证伪+楔子 三件套**
   - "反共识是X，D30验证Y指标，楔子是Z场景"
   - Kill阈值：D30如果Y<阈值，立即停止

2. **学可迁移规律，避不可迁移因素**
   - "Google的推荐奖励机制可以学"
   - "但2007年的AWS成本优势不可迁移"

3. **分数用于排序，验证用于决策**
   - "8.5分排第一，但核心假设需要D30验证"
   - "验证通过再放大投入"
